# トークンエコノミクス 数値設計

**バージョン**: 0.1.1  
**作成日**: 2026-03-03  
**ステータス**: ドラフト（数値はコミュニティ合意で変更可）

> 数値はあくまで初期案。MVP運用後のデータをもとにDAOガバナンスで見直す。

---

## 1. 基本設計方針

### 1.1 トークンの価値の錨（アンカー）

**ユーティリティトークン（以下 YUI: yui Token）は「法定通貨」ではなく「農的労働の30分」を基準単位とする。**

```
1 YUI = 農作業 30分 相当の貢献価値（コミュニティの合意単位）
```

この設計の理由：
- 法定通貨との直接換算を持たないことで資金決済法の規制対象外を目指す
- 「時間」を基準にすることでインフレ耐性が高い
- 農的コミュニティの感覚に合ったスケール感

### 1.2 現金とトークンの完全分離原則

```
【現金の流れ】                    【トークンの流れ】
  寄付（任意）                      貢献活動でのみ発行
  実費の割り勘（光熱費・運営費）      ↕
       ↓                           コミュニティ内で流通
  Treasury（共有財）  ───────→     農機具・農産物・サービスと交換
  （農機具・運営費に使用）
```

**現金はトークンと交換しない。この分離がすべての法的安全性の土台。**

### 1.3 トークンの種類と役割

| トークン | 記号 | 発行方法 | 用途 |
|---------|------|---------|------|
| ユーティリティトークン | YUI | **貢献活動でのみ**発行 | 農産物・サービスとの交換・農機具予約 |
| ガバナンストークン | LMG | **長期貢献でのみ**発行（現金との交換なし） | DAO投票権・提案権 |
| メンバーシップSBT | — | 加入時に発行（無料） | コミュニティ参加資格 |
| ContributionSBT | — | 取引完了時に発行 | 実績バッジ（非売品） |
| Earth Care SBT | — | Earth Care活動承認時 | 環境貢献バッジ（非売品） |

## 2. 現金の扱い：寄付 × 実費割り勘モデル

### 2.1 基本方針

現金はトークンと一切連動しない。コミュニティへの現金参加は以下の2種類のみ。

| 種別 | 内容 | 法的性格 | 金額の目安 |
|------|------|---------|----------|
| **① 共有財への寄付** | コミュニティの農機具・施設整備・文化活動のための任意の寄付 | 贈与（対価なし） | 自由（例：¥1,000〜¥100,000） |
| **② 実費の割り勘** | 共有スペースの光熱費・アプリ運営費・農機具保険料などの実費分担 | 費用分担（対価あり・実費のみ） | 均等割り（例：¥3,000〜¥5,000/年） |

> ✅ **法的安全性の根拠：**  
> - 寄付は対価性ゼロ → 前払式支払手段の「対価の取得」に該当しない  
> - 実費割り勘は「サービスの対価」ではなく「コストの分担」→ 同様に非該当  
> - **どちらもトークンを受け取らない**ため、現金↔トークンの交換関係が存在しない

### 2.2 参加フロー

```
【新メンバーの加入フロー】

メンバーシップSBT の取得（無料・LINEログインのみ）
  ↓
ウェルカムボーナス 10 YUI を受け取る（全員一律・現金不要）
  ↓
任意で以下を選択：
  ┌──────────────────────────────────────┐
  │ A. 共有財に寄付する（任意・任意額）     │
  │    → Treasury に現金が入る             │
  │    → 感謝の意として記録される           │
  │    → トークンとの交換は一切なし         │
  ├──────────────────────────────────────┤
  │ B. 実費を割り勘する（任意・均等額）     │
  │    → 光熱費・運営費の一員として参加     │
  │    → 領収書あり（実費の証明）           │
  │    → トークンとの交換は一切なし         │
  └──────────────────────────────────────┘
  ↓
あとは「貢献活動」だけで YUI を獲得・使用
```

### 2.3 Treasury（共有財）の収支モデル

30名コミュニティ、全員が実費割り勘に参加した場合の試算：

```
【収入】
  実費割り勘  30名 × ¥3,000/年   =   ¥90,000
  有志の寄付  数名から（試算不可）=   ¥50,000 〜
  ──────────────────────────────────
  合計                           ≈  ¥140,000〜/年

【支出（DAO投票で決定）】
  農機具メンテナンス費             ¥50,000
  アプリ・インフラ運営費           ¥40,000
  イベント・祭り・研修費           ¥30,000
  予備費                          ¥20,000
  ──────────────────────────────────
  合計                           ≈  ¥140,000/年
```

### 2.4 LMGとの関係（重要な変更）

現金とガバナンストークンの連動を**廃止**。LMGは「貢献の積み重ね」でのみ取得できる。

| 旧設計（廃止） | 新設計 |
|--------------|-------|
| ¥100,000出資 → LMG 50枚 | 廃止 |
| ¥10,000年会費 → LMG 5枚 | 廃止 |
| **ContributionSBT 累計10枚 → LMG 5枚** | ✅ 維持 |
| **Earth Care SBT 累計5枚 → LMG 10枚** | ✅ 維持 |
| **コミュニティ運営1年継続 → LMG 10枚/年** | ✅ 維持 |

> ガバナンスへの参加権は「お金」ではなく「活動の継続」から生まれる。これがパーマカルチャーのFair Share原則に最も整合する。

## 3. YUI（ユーティリティトークン）の発行レート

### 3.1 活動別 発行レート

コミュニティで合意すべき初期案。

#### People Care（お手伝い・農産物）

| 活動 | 発行量 | 根拠 |
|------|-------|------|
| 農作業のお手伝い | **2 YUI / 時間** | 1 YUI = 30分が基本単位 |
| 農産物の提供（一般野菜） | **1 YUI / 250g** | 直売所の参考価格帯から設定 |
| 農産物の提供（希少品種・在来種） | **2 YUI / 250g** | 希少性プレミアム |
| 種・苗の提供 | **3 YUI / 袋（品種問わず）** | 種の図書館との連動 |
| 農業技術の指導（勉強会） | **4 YUI / 時間** | 専門知識への加重 |
| マッチング仲介・コミュニティ運営 | **1 YUI / 件** | 間接貢献への報酬 |

#### Earth Care（環境再生活動）

Earth Care活動は **+50%のプレミアム** を乗せて発行（環境貢献の優遇）。

| 活動 | 発行量 | 根拠 |
|------|-------|------|
| 堆肥作り | **3 YUI / 時間** | 通常作業 × 1.5 |
| 植樹 | **5 YUI / 本** | 長期的な環境価値 |
| 農薬不使用農業（シーズン認定） | **30 YUI / シーズン** | 継続的な環境貢献 |
| 水源保全活動 | **3 YUI / 時間** | 通常作業 × 1.5 |
| 生物多様性の記録提出 | **2 YUI / 回** | データ貢献 |

## 4. LMG（ガバナンストークン）の発行レート

### 4.1 発行基準

ガバナンストークンは「**長期的なコミュニティへのコミットメント**」に対して発行する。

| 発行トリガー | 発行量 | 頻度上限 |
|------------|-------|---------|
| コミュニティ出資 ¥10,000 | 5 LMG | 初回のみ |
| コミュニティ出資 ¥100,000以上 | 50 LMG | 初回のみ |
| ContributionSBT 累計10枚達成 | 5 LMG | 一度のみ |
| ContributionSBT 累計50枚達成 | 20 LMG | 一度のみ |
| Earth Care SBT 累計5枚達成 | 10 LMG | 一度のみ |
| コミュニティ運営1年継続 | 10 LMG / 年 | 毎年 |

### 4.2 LMGの投票権重み

| 保有量 | 投票権 |
|-------|-------|
| 1〜9 LMG | 1票 |
| 10〜49 LMG | 2票 |
| 50 LMG以上 | 3票（上限） |

> 大口出資者が支配的にならないよう、投票権に上限を設ける（Quadratic Votingも検討）。

## 5. Demurrage（保有税）設計

### 5.1 基本設計

使われないトークンが溜まりすぎることを防ぐため、保有時間に応じてトークンが緩やかに減衰する。

```
Demurrage率: 月1%（年間約11.4%の減衰）

例）
  100 YUI 保有 → 1ヶ月後 → 99 YUI（1 YUIが消却）
  100 YUI 保有 → 6ヶ月後 → 94.1 YUI
  100 YUI 保有 → 1年後  → 88.6 YUI
```

> 参考：地域通貨「Chiemgauer（ドイツ）」は年間8%のDemurrage。

### 5.2 Demurrage の処理先

消却されたトークンは **Treasury に自動移転**（Burn ではなく）。
これにより、使わない人の余剰が自動的にコミュニティ共有財へ再投資される。

```
Demurrage で減衰した YUI
    ↓
Treasury.sol に自動移転
    ↓
DAOガバナンスで使途を投票（農機具購入・イベント・etc.）
```

### 5.3 Demurrage 免除枠

少額保有者への負担を軽減するため、免除枠を設ける。

| 保有量 | Demurrage |
|-------|----------|
| 0〜20 YUI | 免除（生活インフラ分として保護） |
| 21〜100 YUI | 月1% |
| 101 YUI以上 | 月1.5%（投機的な大量保有の抑制） |

## 6. トークン流通シミュレーション（MVP・30名）

### 6.1 前提条件

- 参加者：30名
- 月間タスク取引：60件（1人が月2件依頼 or 受ける）
- 平均タスク時間：2時間
- 月間農産物取引：90件

### 6.2 月間フロー試算

```
【発行（Mint）】
  農作業 (60件 × 2時間 × 2 YUI) =    240 YUI / 月
  農産物提供 (90件 × 平均2 YUI)  =    180 YUI / 月
  Earth Care活動 (10件 × 平均5)  =     50 YUI / 月
  ウェルカムボーナス（月1〜2名加入）=  30 YUI / 月
  ────────────────────────────────
  月間発行合計                   ≈    500 YUI / 月

【消却（Burn → Treasury移転）】
  Demurrage（平均保有50 YUI × 30名 × 1%）= 15 YUI / 月

【流通量の推移】
  6ヶ月後の総流通量  ≈ 500 × 6 - 15 × 6 ≈  2,910 YUI
  1年後の総流通量    ≈ 500 × 12 - 15 × 12 ≈ 5,820 YUI
```

### 6.3 健全性チェック

| 指標 | 目標値 | 判断基準 |
|------|-------|---------|
| 月間アクティブ率 | 70%以上 | 30名中21名以上が月1回以上取引 |
| トークンの平均流通速度 | 月1〜2回転 | 保有トークンが月1〜2回使われている |
| Treasury残高 | 月間発行量の20〜30% | 少なすぎると共有財の維持困難 |
| Demurrage比率 | 月間発行量の5%以下 | 高すぎると「貯めても無駄」感が生まれる |

## 7. 数値見直しのガバナンスプロセス

初期数値は仮設定。以下のサイクルでDAOが見直す。

```
【四半期レビュー】
  1. 管理者ダッシュボードで流通データを確認（SCR-14）
  2. 問題がある指標を特定（上記ヘルスチェック）
  3. 変更案をDAO提案として作成
  4. LMG保有者で投票（1週間）
  5. 可決された場合、次の月から適用

【見直し対象パラメータ】
  - 活動別発行レート（例：「農作業が過小評価されている」）
  - Demurrage率（例：「減衰が速すぎて使う気になれない」）
  - 参加ティアの出資額
  - Earth Care プレミアム率
```

## 8. 法的整合性チェック

| チェック項目 | 本設計の回答 |
|------------|------------|
| YUIは法定通貨で購入できるか？ | ❌ できない（活動でのみ獲得） |
| YUIは法定通貨に換金できるか？ | ❌ できない（コミュニティ内クローズド） |
| 現金を受け取る場面はあるか？ | ✅ ある（寄付・実費割り勘のみ） |
| 現金とトークンは連動するか？ | ❌ 一切しない（完全分離） |
| 発行体は誰か？ | コミュニティDAO（特定個人・法人ではない） |
| 前払式支払手段に該当するか？ | ✅ 非該当の可能性が高い（「対価の取得」によるトークン発行がないため） |
| 暗号資産に該当するか？ | ✅ 非該当の可能性が高い（不特定多数間での支払いでなく、クローズドなコミュニティ内流通） |
| 寄付金の税務処理は？ | 任意団体への寄付として処理。法人格の有無によって異なる |

> ⚠️ **注意**: 上記はあくまで設計上の考え方です。実際の運用開始前に資金決済法・税務の専門家への法的確認を強く推奨します。特に任意団体 vs 法人格の選択は税務・会計処理に影響します。
