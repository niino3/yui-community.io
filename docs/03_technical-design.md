## 1. Web3技術スタック

### 1.1 ブロックチェーン選定

**要件：** 低ガス代 / 高速処理 / 環境負荷が低い

| 候補 | 特徴 | 検討ポイント |
|------|------|------------|
| **Polygon (PoS)** | EVM互換・低ガス代・実績豊富 | 最有力候補 |
| **Avalanche** | サブネットによるカスタムチェーン構築が可能 | スケーラビリティに優れる |
| **Arbitrum / Optimism** | EthereumのL2・セキュリティが高い | やや複雑 |
| **Base** | Coinbase運営のL2・開発者体験が良い | エコシステムの成長に期待 |

> **暫定推奨：Polygon** — EVM互換で開発ライブラリが豊富、ガス代が安く農村コミュニティの小額決済に適している。

### 1.2 スマートコントラクト

- **言語：** Solidity（EVM互換）
- **開発フレームワーク：** Hardhat または Foundry
- **主要コントラクト：**

**マルチテナント基盤（共通）**

| コントラクト | 機能 |
|-------------|------|
| `CommunityFactory.sol` | 新コミュニティの全コントラクトを一括デプロイするファクトリー |
| `CommunityRegistry.sol` | 全コミュニティのコントラクトアドレスを管理するレジストリ |

**コミュニティごとにデプロイされるコントラクト（Factory経由）**

**トークン系**

| コントラクト | 規格 | 機能 | 対応倫理 |
|-------------|------|------|---------|
| `CommunityToken.sol` | ERC-20（Upgradeable） | ユーティリティトークン（有効期限・Demurrage機能付き） | People Care / Fair Share |
| `GovernanceToken.sol` | ERC-20（Upgradeable） | ガバナンストークン（投票権。長期貢献者・運営者へ配布） | Fair Share |

**SBT系（ERC-5192 / 譲渡不可）**

| コントラクト | 規格 | 機能 | 発行タイミング | 対応倫理 |
|-------------|------|------|-------------|---------|
| `MembershipSBT.sol` | ERC-5192 | メンバーシップ証明（居住地・参加資格） | コミュニティ加入時 | People Care |
| `ContributionSBT.sol` | ERC-5192 | Proof of Help（お手伝い・農産物提供の実績バッジ） | 取引完了時 | People Care |
| `EarthCareSBT.sol` | ERC-5192 | Proof of Earth Care（環境再生活動の実績バッジ）+ 検証ロジック | 活動承認時 | Earth Care |

**ロジック系**

| コントラクト | 機能 |
|-------------|------|
| `Escrow.sol` | マッチング成立後の取引エスクロー・自動送金 |
| `Governance.sol` | DAOガバナンス（投票・提案ロジック） |
| `ReputationScore.sol` | 信頼スコアの集計・記録（ContributionSBT連携） |
| `Treasury.sol` | コミュニティ財務管理（余剰トークンの自動受け入れ） |

> マルチテナントの全体設計は [08_multi-tenant-architecture.md](./08_multi-tenant-architecture.md) を参照。

### 1.3 ウォレット・認証

**要件：** 専門知識不要なログイン体験

| 候補 | 特徴 |
|------|------|
| **Magic (Magic Link)** | メールアドレスだけでウォレット生成。MPCベース |
| **Web3Auth** | SNSログイン（Google, LINE等）対応。カスタマイズ性が高い |
| **Privy** | Web2ログインとWeb3ウォレットをシームレスに統合 |

> **暫定推奨：Privy または Web3Auth** — LINEログイン対応が農村コミュニティの普及に有効。

### 1.4 ID・証明（SBT）

- **規格：** ERC-5192（Minimal Soulbound NFT）
- **SBT3種別の証明内容：**

| SBT種別 | コントラクト | 証明内容 |
|---------|------------|---------|
| **メンバーシップSBT** | `MembershipSBT.sol` | 居住地・コミュニティ参加資格・スキル（農業経験など） |
| **Proof of Contribution SBT** | `ContributionSBT.sol` | お手伝い実績・農産物提供実績（Proof of Help）|
| **Earth Care SBT** | `EarthCareSBT.sol` | 堆肥作り・植樹・農薬不使用・水源保全などの環境再生活動実績 |

> `EarthCareSBT.sol` は活動の検証ロジック（コミュニティ承認投票 or オフチェーン写真・GPS記録との連携）を内包する。

## 2. アーキテクチャ設計

### 2.1 オフチェーン ↔ オンチェーンのバランス

農作業中・直売所でのリアルタイム利用を考慮し、すべてをオンチェーンにしない設計を採用する。

```
[ユーザー操作]
    ↓
[オフチェーン層]  ← QRコード決済・チャット・マッチング掲示板
  (高速・低コスト)    ユーザーの操作はまずここで処理
    ↓ バッチ処理 or 重要イベント時
[オンチェーン層]  ← 残高確定・信頼スコア・SBT発行・エスクロー決済
  (信頼性・透明性)
```

**オフチェーンで処理するもの：**
- チャット・通知
- マッチング掲示板の投稿・閲覧
- 小額・頻度の高いトークン送受信の一時記録

**オンチェーンで処理するもの：**
- トークンの最終残高確定
- エスクロー決済（取引完了の記録）
- SBT発行・剥奪
- 信頼スコアの更新
- DAOガバナンスの投票

## 3. フロントエンド技術スタック

| 項目 | 採用技術 | 理由 |
|------|---------|------|
| フレームワーク | Next.js (App Router) | SSR対応・エコシステムが充実 |
| スタイリング | Tailwind CSS | 高速なUI構築 |
| Web3連携 | wagmi + viem | React向けWeb3ライブラリの標準 |
| ウォレット | Privy / Web3Auth SDK | SNSログイン対応 |
| QRコード | `qrcode.react` / `html5-qrcode` | 生成・読み取り両対応 |
| 状態管理 | Zustand | シンプルで軽量 |

---

## 4. バックエンド技術スタック

| 項目 | 採用技術 | 理由 |
|------|---------|------|
| ランタイム | Node.js (Bun) | 高速・TypeScript対応 |
| フレームワーク | Hono | 軽量・エッジ対応 |
| DB | PostgreSQL | リレーショナルデータに最適 |
| ORM | Drizzle ORM | TypeScript親和性が高い |
| キャッシュ | Redis | セッション・オフチェーン残高の一時保存 |
| プッシュ通知 | Web Push / LINE Messaging API | 農村コミュニティにはLINEが有効 |

## 5. 開発ロードマップ

### フェーズ0：プロトタイプ（シングルテナント）

**目標：** 1コミュニティで動作実証（マルチテナント化前の検証）

- [ ] シングルテナントとして全機能をMVP実装
- [ ] 1つのパーマカルチャー教室で試験運用（〜30名）
- [ ] フィードバックをもとに設計を修正

### フェーズ1：MVP（最小実行可能プロダクト）

**目標：** 小規模コミュニティ（〜30人）でのトークン送受信・マッチング基本機能

- [ ] ウォレット接続・SNSログイン実装
- [ ] ユーティリティトークン（ERC-20）コントラクト開発
- [ ] QRコードによるトークン送受信UI
- [ ] マッチング掲示板（依頼投稿・応募）
- [ ] 残高・取引履歴表示

### フェーズ2：信頼・評価システム + Earth Care

**目標：** 初対面でも安心して取引できるレピュテーションシステムと、環境貢献の可視化

- [ ] `MembershipSBT.sol` — メンバーシップSBT発行・管理
- [ ] `ContributionSBT.sol` — Proof of Help SBT発行（取引完了連動）
- [ ] `EarthCareSBT.sol` — Earth Care SBT発行・活動承認ロジック
- [ ] `ReputationScore.sol` — 信頼スコアコントラクト（ContributionSBT連携）
- [ ] `Escrow.sol` — エスクロー機能（取引完了確認後の自動送金）
- [ ] 取引レビュー・評価UI
- [ ] Earth Care活動記録UI（写真・GPS記録のアップロード）

### フェーズ3：経済圏の成熟（Fair Share の実装）

**目標：** 自律的に運営されるDAOコミュニティ（パーマカルチャー「余剰の分かち合い」原則の実装）

- [ ] `GovernanceToken.sol` — ガバナンストークン発行・配布
- [ ] `Governance.sol` — DAOガバナンス（投票・提案ロジック）
- [ ] `Treasury.sol` — コミュニティ財務管理（余剰トークンの自動受け入れ）
- [ ] `CommunityToken.sol` へのDemurrage（保有税）機能追加
- [ ] `CommunityToken.sol` へのトークン有効期限・自動消却機能追加

## 6. セキュリティ考慮事項

- スマートコントラクトの **外部監査**（フェーズ2以降）
- オフチェーン個人情報の **暗号化保存**
- ブロックチェーン上には **匿名化されたDIDのみ** を記録
- レート制限・スパム対策（特にマッチング掲示板）
- マルチシグウォレットによる **コミュニティ資金の管理**
