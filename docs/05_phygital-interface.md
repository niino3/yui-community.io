# フィジカル ↔ デジタル接続層（Phygital Interface）

**バージョン**: 0.1.0  
**作成日**: 2026-03-03  
**ステータス**: ドラフト

> 物理世界（農機具・農作業・作物）とデジタルシステム（トークン・SBT・スマートコントラクト）をつなぐ「接点層」の設計。

---

## 概念マップ

```
【物理世界】                【接点技術】              【デジタル世界】

モノ（農機具など）
  ├─ 農機具（耕運機・草刈機）  ──── QRコード/NFC    ──→  利用予約トークン
  ├─ 共同倉庫・道具           ──── IoTセンサー      ──→  稼働状況・在庫
  └─ 農地・区画              ──── GPS/IoT          ──→  区画利用権NFT

タスク（農作業）
  ├─ 草取り・草刈り           ──── QRコード         ──→  ContributionSBT
  ├─ 田植え・収穫             ──── AIカメラ          ──→  Proof of Help
  ├─ 堆肥作り・植樹           ──── AIカメラ/GPS     ──→  Earth Care SBT
  └─ 農薬不使用の管理         ──── IoT土壌センサー   ──→  Earth Care SBT

食（作物）
  ├─ 収穫物・農産物           ──── QRコード/AI計測  ──→  トークン発行量算出
  ├─ 直売所での取引           ──── QRコード決済      ──→  トークン送受信
  └─ 種・苗の流通             ──── QRコード/NFC     ──→  種の図書館管理
```

---

## 1. モノ（農機具・共有資産）

### 1.1 農機具の共同管理

農機具はコミュニティの共有資産として管理する。物理的なタグ（QR/NFC）と連動して、予約・利用・返却・メンテナンスをトークンで管理する。

#### QRコード / NFCタグ

```
[農機具にQR/NFCタグを貼付]
    ↓
ユーザーがスキャン
    ↓
  ┌─────────────────────────────────────────┐
  │  「予約」: 利用開始日時・時間を入力        │
  │           → Escrow.sol にトークンをロック │
  │  「返却」: 返却確認スキャン               │
  │           → エスクロー解放                │
  │           → ContributionSBT 発行（返却者）│
  └─────────────────────────────────────────┘
```

#### IoT（GPSトラッカー・稼働センサー）

| センサー | 取得データ | 活用方法 |
|---------|----------|---------|
| GPSトラッカー | 農機具のリアルタイム位置 | 盗難防止・返却確認 |
| エンジン稼働センサー | 稼働時間の累積 | メンテナンス時期の自動通知・費用のTreasury自動引き落とし |
| 振動センサー | 異常検知 | 故障の早期発見・メンテナンス依頼の自動生成 |

#### データフロー（農機具）

```
IoTデバイス（GPS・稼働センサー）
    ↓ MQTT / HTTP
バックエンド（オフチェーン処理）
    ↓ 条件トリガー（稼働時間超過、位置異常など）
Oracle（コミュニティ承認 or 自動）
    ↓
Escrow.sol / Treasury.sol（オンチェーン決済）
```

### 1.2 農地区画の利用管理

- 農地の区画をNFT化し、シーズン単位で利用権をトークンと交換
- GPSで区画の境界を定義し、利用状況を記録

## 2. タスク（農作業）

農作業の「始まり」と「終わり」を記録することで、Proof of Help / Proof of Earth Care を発行する。

### 2.1 タスクフロー全体像

```
【依頼者】                              【作業者】
タスクを掲示板に投稿                      掲示板でタスクを発見
トークンをエスクローにロック ────────→   「参加する」をタップ
                                          ↓
                                    作業開始QRをスキャン
                                    （位置情報・時刻を記録）
                                          ↓
                                       作業実施
                                          ↓
                                    作業完了QRをスキャン
                                    （AIカメラで完了証跡撮影）
                                          ↓
依頼者が完了を承認 ←─────────────── 完了通知が届く
トークン自動送金                    ContributionSBT 発行
```

### 2.2 QRコード（タスク開始・完了）

現場に設置するQRコードまたは依頼者のスマホ画面のQRをスキャンする。

| QRの種類 | 設置場所 | スキャン者 | 記録内容 |
|---------|---------|----------|---------|
| タスク開始QR | 作業現場 / 依頼者のスマホ | 作業者 | 開始時刻・位置情報・作業者ウォレットアドレス |
| タスク完了QR | 依頼者のスマホ | 作業者 | 完了時刻・AIカメラ撮影データのハッシュ |

### 2.3 AIカメラ（作業証明）

スマートフォンのカメラを活用して作業の証拠を記録・分析する。

| 用途 | AIの機能 | 精度の担保方法 |
|------|---------|--------------|
| 草取り・草刈り完了証明 | 作業前後の草の量を比較分析（画像差分） | コミュニティ承認との組み合わせ |
| 収穫量の計測 | 収穫物の量・品質を画像から推定 | 重量計IoTとのダブルチェック |
| 堆肥の量証明 | 堆肥の体積・状態を推定 | 複数写真からの3D推定 |
| 植樹本数カウント | 植えた苗木の本数を自動カウント | GPS位置データとの照合 |

> **実装方針：** MVP段階ではAI分析はオフチェーンで処理し、結果のハッシュのみをオンチェーンに記録。AIモデルは段階的に精度向上。

### 2.4 Earth Care タスク専用フロー

```
Earth Care活動（堆肥作り・植樹・農薬不使用など）
    ↓
スマホアプリで記録開始（GPS・タイムスタンプ）
    ↓
作業中：写真・動画・IoTデータ（土壌センサーなど）を収集
    ↓
作業完了：AIカメラで証跡撮影 → データをバックエンドへ送信
    ↓
コミュニティ承認フロー（3名以上の承認で成立）
    ↓
Earth Care SBT 発行 + ユーティリティトークン付与
```

## 3. 食（作物・農産物）

### 3.1 トレーサビリティQR（生産者 → 消費者）

農産物にQRタグをつけることで、生産情報の透明性を確保する。

```
[農産物にQRタグを貼付]
    ↓
消費者がスキャン
    ↓
  ┌────────────────────────────────────────────┐
  │  表示情報:                                  │
  │  - 生産者名・SBT実績（信頼スコア）          │
  │  - 生産地（GPS座標）                        │
  │  - 栽培方法（農薬不使用、有機など）          │
  │  - 収穫日                                   │
  │  - Earth Care認定マーク（該当する場合）      │
  └────────────────────────────────────────────┘
    ↓
トークンで購入 → QRコード決済
```

### 3.2 収穫量のAI計測とトークン発行量の自動算出

AIカメラ + IoT重量計を組み合わせて、農産物の量・品質からトークン発行量を自動計算する。

| ステップ | 技術 | 処理内容 |
|---------|------|---------|
| 収穫物の撮影 | AIカメラ（スマホ） | 品目・量・品質グレードを推定 |
| 重量計測 | IoT重量計（オプション） | 実重量でAI推定値を補正 |
| トークン量算出 | バックエンド | 品目・量・品質グレードからトークン量を計算 |
| 承認・発行 | コミュニティ承認 or 自動 | ユーティリティトークンをMint |

**トークン換算レート例（要コミュニティでの合意）：**

| 品目 | 単位 | トークン量（例） |
|------|------|---------------|
| 野菜（一般） | 1kg | 10 TOKEN |
| Earth Care認定野菜 | 1kg | 15 TOKEN（+50%プレミアム） |
| 種（希少品種） | 1袋 | 20 TOKEN |
| 果物 | 1kg | 12 TOKEN |

### 3.3 種の図書館

種の貸し借りをQRコードで管理する。

```
[種の袋にQRタグ]  ─→  スキャンで借り出し記録
                       （トークン or デポジット）
                  ─→  収穫後に種を返却 + 増量（借りた量 + 20%）
                       ContributionSBT 発行
```

## 4. Oracle設計（物理データ → オンチェーン連携）

物理世界のデータをオンチェーンに安全に持ち込むためのOracle（データ橋渡し）設計。

### 4.1 Oracleの種類と選択方針

| Oracle種別 | 仕組み | メリット | デメリット | 採用フェーズ |
|-----------|-------|---------|----------|------------|
| **コミュニティOracle** | メンバーが多数決で承認 | パーマカルチャー的・低コスト | 時間がかかる・人手依存 | MVP〜全フェーズ |
| **バックエンドOracle** | サーバーがAI分析結果を署名してオンチェーンに送信 | 自動化・高速 | 中央集権的リスク | フェーズ2〜 |
| **分散型Oracle（Chainlink等）** | 外部の分散型Oracleネットワーク | 改ざん耐性が高い | コスト・複雑度が高い | フェーズ3〜 |

### 4.2 コミュニティOracle（推奨）の詳細フロー

パーマカルチャーの「相互監視・信頼」に最も合致するアプローチ。

```
作業者が証跡データ（写真・GPS・IoTデータ）を提出
    ↓
バックエンドがデータのハッシュを計算 → オフチェーンに保存
    ↓
コミュニティメンバー（3名以上）に承認リクエストを通知
    ↓
承認者がデータを確認 → アプリ上で「承認」or「否認」
    ↓
多数決（2/3以上）で承認成立
    ↓
バックエンドがスマートコントラクトに承認結果を送信
    ↓
SBT発行 + トークンMint
```

### 4.3 データの改ざん防止

| データ種別 | オフチェーン保存 | オンチェーン記録 |
|-----------|---------------|---------------|
| 写真・動画 | バックエンドサーバー（暗号化） | コンテンツハッシュ（SHA-256）のみ |
| GPS座標 | バックエンドDB | 作業場所のハッシュのみ |
| IoTセンサーデータ | 時系列DB | 集計値のハッシュ |
| 個人情報 | バックエンドDB（オフチェーン） | 匿名DIDのみ |

## 5. 段階的導入ロードマップ

### フェーズ1（MVP）：QRコードのみ

**コスト：** 低（スマホだけで完結）

- タスク開始・完了のQRスキャン
- 農産物のQRトレーサビリティタグ
- 農機具予約のQR
- コミュニティ承認（チャット/アプリ上）

### フェーズ2：AIカメラ導入

**コスト：** 中（スマホ標準カメラを活用・専用ハードウェア不要）

- 作業証明写真のAI自動分析
- 収穫量の画像計測
- Earth Care活動の証跡撮影

### フェーズ3：IoT本格導入

**コスト：** 高（ハードウェア調達・設置・メンテナンスが必要）

- 農機具GPSトラッカー
- 土壌センサー（農薬不使用証明）
- IoT重量計（収穫量精密計測）
- 環境センサー（気温・湿度・日照）

## 6. 技術的課題と対策

| 課題 | 詳細 | 対策 |
|------|------|------|
| **電波が届かない圃場** | 山間部ではモバイル回線が不安定 | オフライン対応モード（ローカルキャッシュ → 電波回復後に同期） |
| **高齢者のスマホ操作** | QRスキャン・アプリ操作への不慣れ | 超シンプルUI・LINEミニアプリ化・家族・若手メンバーによるサポート体制 |
| **AIカメラの精度** | 農作物・作業状況の認識精度にばらつき | コミュニティ承認とのハイブリッド運用（AI＋人力確認） |
| **IoTデバイスの管理** | 電池切れ・故障・紛失 | コミュニティ共有管理台帳 + メンテナンス費用をTreasuryから自動拠出 |
| **データの改ざん** | 虚偽の作業報告 | ハッシュのオンチェーン記録 + コミュニティ承認 + 信頼スコアへのペナルティ |
| **プライバシー（GPS追跡）** | 位置情報の取り扱い | 個人を特定できる詳細位置はオフチェーン保存、オンチェーンは区画IDのみ |
