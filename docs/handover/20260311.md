## 10. メモ・注意事項

### 設計上の決定事項

- **communityId は 1 から開始**: mapping のデフォルト値 0 と区別するため
- **Factory が一時的に owner**: mint/issue 後に admin へ ownership 移譲
- **テナント解決の優先順位**: ヘッダー > サブドメイン > クエリ
- **role による認可**: member < operator < admin

### 既知の課題

- ルーティング設定（api.php）にミドルウェア適用がまだ
- テストネットへの Factory/Registry デプロイがまだ

## 11. 今日の学び

- Solidity の mapping は0がデフォルト値 → ID を 1 から開始する設計パターン
- Factory パターンで ownership を移譲するタイミングが重要
- Laravel のミドルウェアで `app()->instance()` を使うとアプリ全体で利用可能
- マルチテナント設計はテナント解決の優先順位が重要
